# Участие в разработке

## Принцип проекта

HydraBridge поддерживает существующее поведение HR Neo. Новая функциональность
не должна незаметно создавать отдельную модель маршрутизации.

## Проверки

Перед pull request:

```sh
make ci
make lint
make smoke-local
```

При изменении интеграции с роутером дополнительно выполните подходящие профили
из [docs/RELEASE.md](docs/RELEASE.md).

## OpenAPI

[api/openapi.yaml](api/openapi.yaml) является источником истины для публичных
структур API. После изменения схем запускайте:

```sh
make generate
```

## Конфиги HR Neo

Точечные helpers должны сохранять посторонние байты конфигов. Bulk replacement
допустим только для endpoint, которые явно документированы как полная замена.
