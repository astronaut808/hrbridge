# HydraBridge

[English](README.en.md)

HydraBridge - единый HTTP API управления для
[HydraRoute Neo](https://github.com/Ground-Zerro/HydraRoute).

Это небольшой агент для роутеров Keenetic с Entware. Он запускается рядом с
`hrneo`, но не маршрутизирует трафик самостоятельно. HydraBridge предоставляет
приложениям, ботам, CLI-инструментам и новым web-интерфейсам стабильный API для
настройки и диагностики возможностей HR Neo.

## Принципы

- `hrneo.conf`, `domain.conf` и `ip.list` остаются источником истины.
- HydraBridge не создает вторую модель маршрутизации поверх HR Neo.
- Удобные сгруппированные ответы API являются read-only представлениями.
- Точечные изменения сохраняют посторонние байты файлов конфигурации.
- Перед изменениями создаются резервные копии.
- Конкурентное редактирование защищено через `ETag`, `X-Config-Revision` и
  `If-Match`.

## Возможности

- Bearer-аутентификация и ротация токена.
- Статус, Doctor и проверка совместимости.
- Raw и structured API для конфигов HR Neo.
- Точечное редактирование domain, GeoSite, CIDR и GeoIP-правил.
- Targets, `PolicyOrder`, DirectRoute и live runtime inspection.
- GeoData, import/export, диагностика доменов и IP.
- Резервные копии, восстановление, audit log и lifecycle HR Neo.
- OpenAPI-контракт для генерации клиентов.

## Границы проекта

HydraBridge поддерживает то, что уже реализовано HR Neo. Управление
VPN-профилями Keenetic, составом подключений внутри policy, failover апстримов,
XRay, подписками и мониторингом доступности не относятся к HR Neo и не входят в
область ответственности HydraBridge.
